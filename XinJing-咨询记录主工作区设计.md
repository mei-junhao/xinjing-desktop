# 心镜 · 咨询记录主工作区 设计文档（吞并 来访者 / 会话 / 报告中心）

> 目标版本：v1.8.0（建议单独成档，先方案 + 评审 ≥95 再动手）
> 取代：`clients.html` + `clients.js`、`session.html` + `session.js`、`reports.html` + `reports.js`
> 设计语言：方向 A 静谧留白（共用 `tokens.css`），与「AI 督导通用取向版」同族

---

## 一、目标与定位

「咨询记录」是咨询师**每天最主要的工作空间**，把当前三个割裂的页面合并为一个**三栏主工作区**：

- 左栏：来访者列表（多状态）
- 中栏：选中来访者的工作区（资料 + 会谈时间线 + 报告矩阵）
- 右栏：针对「某来访 × 某次咨询」的**深度文字整理编辑器**（逐字稿 / SOAP / DAP / 反思 + AI 坞）

报告中心不再是独立页面，而是中栏的一个**视图/子标签**，与编辑流无缝打通。

---

## 二、现有三页的问题

| 页面 | 问题 |
|------|------|
| `clients.html` | 只是列表，点进去没有连续工作流；与记录编辑割裂 |
| `session.html` | 单条记录孤立编辑，无来访者上下文、无相邻会谈对照 |
| `reports.html` | 矩阵孤立，看得到产出却不能直接编辑；跨页跳转打断心流 |

## 四、各栏详细设计

### 4.1 左栏 · 来访者 rail
- 搜索框（姓名/标签/备注，`clients.html:21` 同款）。
- 状态筛选 pill：`全部 / 咨询中 / 暂停 / 已结束`（`clients.html:25-28`）。
- 客户卡：状态圆点（绿=咨询中/黄=暂停/灰=已结束）、姓名、末次会谈日期、节数、欠费标记（复用 `aggregateClient`）。
- 点击客户卡 → 中栏加载该来访者工作区；右栏清空待选。
- 底部「+ 新建来访者」（弹 `clients.html` 编辑模态，保留）。

### 4.2 中栏 · 来访者工作区
- **来访者头**：姓名、状态徽章、标签、关键统计（节数 / 首末次日期 / 应收·已收·欠费，来自 `aggregateClient`/`aggregateAll`）；快捷按钮「+ 新建咨询记录」「导出」。
- **子标签**：
  - `时间线`：该来访者全部会谈（`Store.getSessionsByClient`），按日期倒序；每行显示日期、第几节、费用/已收、是否有逐字稿/SOAP/督导标记；点击 → 右栏打开该次咨询的深度编辑器。
  - `报告矩阵`：**吞并 reports.html**——行=节次、列=逐字稿/SOAP/DAP/反思/个体督导/团体督导（`reports.html:62-72`）；点击任意单元格直接跳到右栏对应 tab 编辑（不再是只读矩阵）。勾选行可「打包导出」（复用 `export.js`）。
- **全站报告矩阵视图**：顶部切到「全站报告矩阵」时，中栏变为跨所有来访者的矩阵（原 reports 行为），筛选联动来访者/报告类型/督导类型。

### 4.3 右栏 · 深度文字整理编辑器
- 顶部：当前「来访者 · 第 N 节（日期）」；未选会谈时显示空态引导。
- **选项卡**（复用 `session.html:200-206` 的 tab 体系，但改为更顺手的布局）：
  - `逐字稿`：Word 式 `contenteditable` 大编辑区 + 选区浮动格式工具栏（与 AI 督导预览同款）。
  - `SOAP`：`S/O/A/P` 四块（`session.html:218-236`）。
  - `DAP`：`D/A/P` 三块。
  - `反思`：自由文本。
  - `AI 坞`：常驻可拖拽侧坞（见 4.4），不占主编辑区。
- 底部保存栏：`标记为已确认` + `保存`（复用 `session.js saveSession`）。

### 4.4 AI 坞（与 AI 督导通用取向版复用）
- 取向分段 + 模板 chip（温尼科特/精神分析/CBT/人本/存在/通用 + 会员自定义）——**直接复用「AI 督导通用取向版」的坞组件与 `Supervisors` 多取向模板表**。
- 动作：`生成整体印象/报告（插入文档）`、`就选区深化`、`总结选区`、`润色选区`、`就此议题开启大师对话`、`整理真人督导录音稿`。
- 结果直接插入右栏当前 tab 的光标处（与督导坞一致）。
- 可拖拽调宽（260–560px）。

## 五、报告中心如何被吞并

1. **结构复用**：原 `reports.html` 的矩阵变成中栏「报告矩阵」子标签 + 「全站报告矩阵」顶部视图，UI/筛选逻辑原样迁移到新页。
2. **从只读→可编辑**：矩阵单元格点击直达右栏对应 tab，消除"看得到改不了"。
3. **导出入口保留**：中栏矩阵勾选 → `exportSelected()`（复用 `export.js`）。
4. 旧 `reports.html` / `reports.js` 在合并完成后**下架**（侧边栏「报告中心」改为本页的矩阵视图入口）。

## 六、数据模型映射（Store 现有字段，无需改结构）

- 来访者：`clients`（id, name, gender, status, tags, notes, billing, createdAt）
- 会谈：`sessions`（id, clientId, date, sessionNumber, durationMinutes, billing{fee,paid}, transcript, soap{s,o,a,p}, dap{d,a,p}, reflection, confirmed）
- 督导：`supervisions`（clientId, type, supervisorName, date, content, conclusion, sessionIds）
- 聚合：`aggregateClient` / `aggregateAll`（store.js 已有，直接复用）

## 八、验收标准

- [ ] 左栏按状态筛选来访者，卡片显示节数/末次/欠费。
- [ ] 点客户 → 中栏显示其资料 + 会谈时间线 + 报告矩阵子标签。
- [ ] 点时间线某次会谈 → 右栏打开该次深度编辑器（逐字稿/SOAP/DAP/反思）。
- [ ] 报告矩阵单元格点击直达右栏对应 tab 编辑（原只读矩阵变为可编辑）。
- [ ] 顶部「全站报告矩阵」视图保留跨来访者矩阵 + 勾选导出。
- [ ] 右栏 AI 坞取向自选 + 模板 + 生成/深化/润色/大师对话/整理录音稿，结果插入文档。
- [ ] 右栏坞可拖拽调宽；保存栏「标记已确认 + 保存」生效。
- [ ] 旧三页下架，侧边栏仅剩「咨询记录」入口。
