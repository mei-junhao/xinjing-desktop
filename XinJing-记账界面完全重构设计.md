> 状态：设计稿，待确认后实施
> 目标版本：1.0.14（与下次发版一起上）
> 约束：保留所有数据契约（`localStorage['psyc_billing_v8']` 的 schema + ID + 函数名），保证 `billing-sync.js` 翻译层与心镜 Store(IndexedDB) 同步不破。

## 〇、为什么现在的版本"用起来不舒服"

不是单点问题，是**结构层**的问题——把它拆成三层：

| 层级 | 当前问题 | 视觉证据 |
|---|---|---|
| **架构层** | 三大区（档案 / 录入 / 明细）单列纵向堆在同一滚动条，**录入与核对被长滚动反复割裂**。iframe 高度只有 ~720px（`billing-shell.html` 限 `height: calc(100vh - 138px)`），左栏 sticky 一旦超过视口就会盖住右栏 | 1.0.13 P0 已尝试两栏 sticky，但在固定高度 iframe 内不可靠 |
| **视觉层** | 用的是 skeuomorphic 蓝色玻璃风（`#0ea5e9` / `backdrop-filter: blur` / `rgba(255,255,255,0.93)`），与心镜主程序「编辑式奢华·暖棕陶土」**完全两套语汇**。其他人看到的"心镜·报告中心"、"心镜·来访者"是衬线字、双层卡、暖纸面、克制的饱和度——一进记账页就跳到 SaaS 后台 | 令牌不一致（`#0ea5e9` vs `--accent: #9E5A3C`）、字体不一致（Inter vs Noto Serif SC）、卡片样式不一致（玻璃模糊 vs `--shadow-bezel`） |
| **交互层** | 字段堆叠在 3-4 列 grid 中没层级；状态标签、徽章、操作按钮视觉权重相近；统计数字与明细里的数字字号、字重不一致，**眼睛需要重新适应** | `.stat-value 18px` vs `.sub-table td 13px`、`.status-tag` 与 `.action-btn` 同字号 |

## 一、整体方案：顶部 Tab 切换 + 单一工作区

> **核心理念**：咨询师记账是**串行任务流**（先选人 → 录一次 → 看统计 / 找账），不是同时看多份报表。把"录入 / 明细 / 统计"做成三个**同级工作区**，通过顶部 tab 切换，进入即看见当前任务所需的全部字段，离开即交还全部视口。

### 1.1 全局布局（iframe 720px 视口内）

```
┌─────────────────────────────────────────────────────────┐
│  ◆ 心镜 · 记账                              [导入旧数据] │ ← 顶部品牌条（贴边 24px 高）
├─────────────────────────────────────────────────────────┤
│  [ 录入  ]   [ 明细  ]   [ 统计  ]    ● 2025-04  ▾      │ ← Tab + 月份切换
├─────────────────────────────────────────────────────────┤
│                                                          │
│             ┌──────────────────────────────┐            │
│             │                              │            │
│             │       当前 Tab 的工作区       │            │
│             │     （单一可滚动容器）         │            │
│             │                              │            │
│             │                              │            │
│             └──────────────────────────────┘            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

- **顶部 24px** 品牌条：左 心镜 mark + 名字 + "记账"小字；右 「导入旧数据」+ 「帮助」icon 按钮。
- **顶部 48px Tab 栏**：三个 tab 胶囊 + 月份选择器。**关键**——tab 是用"内容身份"切的，不是装饰。tab 切换时**整块工作区内容替换**，不挤在同一屏。
- **工作区**：单一可滚动容器，最大宽度 880px，水平居中，padding 32px 40px。

### 1.2 为什么不用"两栏布局"

| 方案 | 在 720px iframe 内的表现 | 评分 |
|---|---|---|
| **顶部 Tab 切换**（推荐）| 三个 tab 各占 100% 视口，互不抢空间；切换有明确视觉反馈；和心镜主程序「页面=身份」的设计哲学一致 | ⭐⭐⭐⭐⭐ |
| 两栏（左 sticky / 右 scroll）| 左栏内容一多就顶住右栏，sticky 在小高度失效；问过 1.0.13 P0 已踩 | ⭐⭐ |
| 上下分屏（录入固定上 / 明细下）| 录入表单展开后挤压明细区，又回到"滚动割裂" | ⭐⭐ |

## 二、视觉令牌：与心镜主程序同源

把当前 iframe 内联的令牌替换为与 `app/css/style.css` 完全同源的版本（**只是复制，不引入新视觉**）。这解决"格格不入"。

  --sidebar: #F4EDE3;      --shell: #F4EDE3;
  --ink: #2A2420;          --ink-2: #6F655C;     --ink-3: #9A8F82;
  --hair: #ECE3D8;         --hair-strong: #DCCFBE;

  /* 主题：暖棕陶土（与主程序一致） */
  --accent: #9E5A3C;       --accent-2: #874A2E;
  --accent-soft: #F1E4DA;  --accent-line: #C99A82;

  /* 语义色：暖调去饱和 */
  --green: #6E7E62;   --green-soft: #ECEFE6;
  --orange: #BC8A4E;  --orange-soft: #F6ECDD;
  --red: #B06A62;     --red-soft: #F5E8E6;
  --blue: #7C93A0;    --blue-soft: #E9EFF2;

  /* 阴影 / 圆角 / 字体 */
  --shadow-sm: 0 1px 2px rgba(42,36,32,.05);
  --shadow:    0 1px 2px rgba(42,36,32,.04), 0 12px 30px rgba(42,36,32,.06);
  --shadow-bezel: 0 0 0 6px var(--shell), 0 0 0 7px var(--hair-strong), 0 12px 30px rgba(42,36,32,.05);
  --r-card: 16px;  --r-ctl: 12px;  --r-sm: 9px;  --r-btn: 999px;

- 标题字体用衬线（与主程序 `.section-title` 一致），数字用等宽。

## 八、风险与验证

### 8.1 风险

| 风险 | 缓解 |
|---|---|
| iframe 高度 720px 可能装不下新 tab 的复杂布局 | **降级**——tab 内容溢出时**整个工作区滚动**（已设计如此），tab 栏固定 48px 不动 |
| 趋势图自绘 SVG 状态多 | 提取 30 行小函数，所有视觉常量走 CSS 变量 |
