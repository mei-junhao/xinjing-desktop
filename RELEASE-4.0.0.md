# 心镜 XinJing 4.0.0 发布说明

> 发布日期：2026-07-16 · 里程碑版本
> 包名：xinjing-desktop · 通道：COS 自动更新（Windows）+ GitHub Releases（macOS，tag v4.0.0 触发 CI）

## 核心变更：产品模型升级为三档会员制

4.0.0 是一次产品定位的重大升级。心镜从「激活 / 受限二态」转为**三档会员体系**，并把「产品访问」与「AI 算力」彻底解耦：

| 档位 | 手工执业工作流 | 资料检索 | AI 临床能力 | 导出 / 皮肤 |
| --- | --- | --- | --- | --- |
| **免费版** | 全功能（来访者 / 督导 / 账务 / 日历不限数量） | 关键词检索 · 100 篇 · 2K 上下文 | 未解锁 | 带水印导出 / 临床皮肤 |
| **会员** | 全功能 | 向量语义检索 · 500 篇 · 4K 上下文 | AI 记录 / 分析 / 报告 / 督导 / 大师会诊 / 思维导图 / 成长轨迹 等 | 无水印导出 / 会员皮肤 |
| **旗舰版** | 全功能 | 向量 + Rerank 精排 · 无限 · 16K 上下文 | 含会员全部 + 自定义督导师 / 深度个案模式 | 全部皮肤 + Rerank |

## 关键功能点

### 1. 取消免费版临床数据硬上限
- **移除**原「受限模式」下 `LICENSE_CAP_CLIENT = 5`、`LICENSE_CAP_SUPERVISION = 50` 的硬性数量锁，以及 `Store.licenseGuard` 抛错拦截。
- 手工执业工作流（来访者管理、督导记录、账务、日历）在**所有档位**均完整可用，不再按数量截断或只读。
- 权益门控收敛到「功能边缘」：AI 工具、无水印导出/打印、高级检索三处统一校验，其余临床数据零限制。

### 2. 统一设计语言系统（新增 `app/css/xj-ui-system.css` + `app/js/entitlements.js`）
- 以 CSS 自定义属性（token）驱动的共享皮肤：
  - **01 临床（clinical）** — 青绿医疗色，所有用户默认可用；
  - **04 安静剧场（theatre）** — 暖红剧场色，会员及以上；
  - **05 夜间观测（observatory）** — 深色观测台，会员及以上。
- 旧皮肤值 `calm` / `xinjing` / `editorial` 首次启动自动迁移到 `clinical`，避免回退到未定义皮肤。
- 会员皮肤受 `premium-skins` 权益门控，非会员切换时提示并引导到方案对比。

### 3. 权益引擎集中化（`entitlements.js`）
- `FEATURE_MIN_TIER` 以 feature key 声明每个功能的最低档位；未知 key 默认拒绝（防御性）。
- `RAG_POLICY` 按档位分文档上限 / 检索方式 / 上下文 token / recall / rerank。
- `effectiveTier()` 统一计算：已激活取 tier；AI 试用中视为旗舰；其余为免费版。
- 主进程（main.js）自行从授权状态计算 RAG 策略，**不再信任渲染页传入的 tier**，免费版触发向量检索时返回 `membership-required`。

### 4. 激活页 / 设置页升级
- 激活页（`activation.html`）内置完整三档对比（100 / 500 / 16,000、向量+Rerank、多大师会诊等）。
- 设置页（`settings.html`）新增「当前方案摘要」+「查看方案」入口 + 三皮肤切换器。

### 5. 加载架构稳健性
- `app.js` 移除运行时 `injectGlobalScripts` 动态注入（旧逻辑用 `typeof` 守卫防重复 const 声明，仍有竞态隐患）。各业务页改为**静态按序加载**共享运行时（store / entitlements / ai / agent / xinjing-chat / app），顺序固定、各文件仅加载一次，消除重复顶层 `const` 声明冲突。
- 全局点击拦截器：访问会员功能页（逐字稿引导 / 督导思维导图 / 真人督导 AI / 成长轨迹 / 账单月历 / 大师会诊）若未授权，自动拦截并引导到方案对比。

### 6. 文案与体验
- 「受限模式」统一改称「免费版」，提示语从「仅可管理前 5 位来访者」改为「手工执业工作流可继续使用；AI 临床能力、无水印导出与高级检索需升级会员」，语气更友好、不再制造恐慌。
- 移除免费版全屏水印与来访者/督导页的上限提示条，仅保留无水印导出/打印拦截。

### 7. 激活窗口放大
- 激活窗口由 520×620 放大到 960×760，承载三档对比表更从容。

## 修复与清理
- 移除已不存在的旧 `license.js` 引用。
- 22 个 HTML 路由全部接入统一 UI 体系（自测 `v4.0.0-4` 强制校验）。
- 自测套件新增 6 项 4.0.0 用例（权益分级 / RAG 策略 / 三皮肤 token / 全路由接入 / 激活设置页 / 主进程策略），全量 **255/0 通过**。
