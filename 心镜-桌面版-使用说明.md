# 心镜 XinJing · 桌面版（EXE）使用说明

## 一、这是什么

心镜原本是一套纯前端（HTML/CSS/JS + IndexedDB）的心理咨询本地管理工作台，运行在浏览器里。
现在已用 **Electron** 把它封装成标准 Windows 桌面程序：双击即用，无需浏览器、无需联网（AI 功能除外），所有数据存在本机。

- 程序名：心镜 XinJing
- 版本：1.0.0
- 安装包：`心镜-XinJing-setup-1.0.0.exe`（NSIS 安装版）
- 安装位置：默认 `C:\Users\<你>\AppData\Local\心镜 XinJing\`（可选其他目录）

## 二、安装与启动

1. 双击 `心镜-XinJing-setup-1.0.0.exe`。
2. 按向导选择安装目录 → 下一步 → 完成。
3. 桌面与开始菜单都会生成「心镜 XinJing」快捷方式，双击打开。
4. 首次启动会看到工作台首页，**数据为空**，逐页录入即可。

> ⚠️ **关于 Windows 拦截提示**：本程序未向微软购买代码签名证书，首次运行时 Windows SmartScreen / 杀毒软件可能弹出"未知发布者"警告。
> 这是正常现象，点击「更多信息 → 仍要运行」即可。**数据完全在本机，不上传任何服务器。**

### 2.5 绿色免安装版（便携版）

除安装版外，另提供 **绿色免安装版**：一个解压即用的文件夹，无需安装、不写注册表、不创建快捷方式，适合 U 盘携带或不想安装的用户。

- 文件夹：`dist/win-unpacked/`（内含 `心镜 XinJing.exe`）
- 压缩包：`dist/心镜-XinJing-绿色免安装版.zip`

使用方式：

1. 把整个 `win-unpacked` 文件夹（或解压 zip）复制到任意位置（如 `D:\心镜\`）。
2. 双击 `心镜 XinJing.exe` 即可运行，功能与安装版完全一致。
3. 数据仍存于本机 `%APPDATA%/XinJing/`，与安装版**共用同一份数据**，二者二选一即可。

> 绿色版未做代码签名，首次运行同样会触发 SmartScreen「未知发布者」提示，点「仍要运行」即可。
> 绿色版不创建桌面/开始菜单快捷方式，也不注册开机自启——若需要这些，请使用安装版。

## 三、激活与试用（90 天免费试用）

心镜桌面版内置**离线激活**，不需要联网服务器。

### 3.1 三种运行状态

| 状态 | 触发条件 | 用户看到的现象 |
|------|----------|----------------|
| 试用（trial） | 首次启动起 90 天内、未激活 | 顶部出现陶土色横幅「试用版 · 剩余 N 天」，**功能全开** |
| 受限（limited） | 超过 90 天、仍未激活 | 顶部红色横幅提示 + **全屏水印**「心镜 · 未激活」+ **禁用导出/打印**（仍可正常使用其余功能） |
| 完整（full） | 已输入有效激活码 | 无任何横幅/水印，导出打印解锁 |

### 3.2 激活步骤

1. 点击任意状态下的横幅「激活」按钮（受限模式下也在顶部）。
2. 在弹出的激活窗口粘贴开发者提供的激活码（格式 `XJ-XXXX-XXXX-…`）。
3. 点「激活」→ 离线校验通过即成功，窗口自动重启，水印/横幅消失。
4. 激活信息只保存在本机 `%APPDATA%/XinJing/license.json`，**不上传任何服务器**。

> 换电脑 / 重装系统后，license.json 不随程序走，需重新激活（用同一个激活码即可，码与"授权给谁"绑定，不绑定机器）。

### 3.3 开发者如何出激活码（genkey）

工程根目录的 `genkey.js` 用于生成激活码：

```bash
cd D:/xinjing-electron
node genkey.js "客户邮箱或姓名"
# 例：node genkey.js "zhangsan@clinic.com"
# 输出：XJ-ORSX-G5CA-... 并记录到本地 keys.log（不进仓库）
```

- 激活码把"客户标识"编进码里，激活后程序界面会显示「授权给：xxx」。
- 主密钥（SECRET）在 `license-core.js` 内，与出码脚本共用；**请勿外泄该文件**，泄露等同可伪造任意激活码。

## 四、重要：桌面版与浏览器版的数据是隔离的

桌面程序自己有一套 IndexedDB 存储，**与你在浏览器（localhost）里跑过的心镜互不相通**。

- 浏览器里的旧数据 **不会** 自动出现在桌面版。
- 迁移做法：
  1. 浏览器版「设置 → 数据管理 → 导出备份」下载 `心镜备份_日期.json`；
  2. 桌面版「设置 → 数据管理 → 导入备份」载入该文件。
- 等价于现有的备份/恢复流程，无需额外开发。

同理，桌面版之间（不同电脑）的数据也不互通，换机同样走"导出 → 导入"。

## 五、桌面专属行为

| 功能 | 说明 |
|------|------|
| 单实例 | 重复双击不会开多个窗口，只会把已开的窗口提到最前。 |
| 最小化到托盘 | 点窗口右上角关闭（×），程序**不会退出**，而是缩到右下角托盘；托盘图标右键「退出」才真正关闭。 |
| 开机自启 | 安装后默认开机自动启动（仅当前用户）。如不需要可手动在「任务管理器 → 启动」里禁用。 |
| AI 功能 | 需在「设置」里填外部 API 地址与密钥；该请求走外网，其余功能纯本地。 |

## 六、给开发者的重打包与发布指引（更新 / 自动更新）

工程在 `D:/xinjing-electron/`，全部依赖与缓存都在 D 盘，**不占用 C 盘系统目录**。

### 6.1 出新版安装包（推荐：GitHub Actions 自动发布）

本项目已配置 `.github/workflows/release.yml`：**打 tag 并推送即自动在云端 Windows 构建并发布**，无需你本机装 Node / Electron（本机受网络或环境限制时尤其推荐这条路）。

```bash
cd D:/xinjing-electron

# 1) 更新前端代码（app/、main.js、preload.js 等）

# 2) 改版本号：编辑 package.json 的 version（如 1.0.0 → 1.0.1）

# 3) 提交代码并打 tag（tag 必须以 v 开头，且对应版本号）
git add -A
git commit -m "release: v1.0.1"
git tag v1.0.1
git push origin main
git push origin v1.0.1        # ← 推送 tag 触发 GitHub Actions 自动构建发布
```

推送 tag 后，GitHub 自动在 Windows runner 上完成：
- `npm ci` 安装依赖（已配 npmmirror 镜像加速 Electron 二进制下载）
- `npm run dist -- --publish never` 打包 **nsis 安装版 + portable 绿色版**（先不自动发布，文件名交给下一步精确控制）
- `node scripts/postbuild.js` 把安装包重命名为中文资产名（`心镜-XinJing-setup-${version}.exe` / `心镜-XinJing-portable-${version}.exe`），并生成 `latest-portable.yml` 与 portable 的独立 blockmap（绿色版增量更新所需）
- `gh release create` 精确上传中文名安装包 / 便携版 / 各自的 `.blockmap` / `latest.yml` / `latest-portable.yml`（带 `--latest`）
- 客户端 `autoUpdater` 随即能检测到新版本

> 你本机只需能 `git push` 到 GitHub 即可，不需要 Node / Electron 环境。
> 在仓库的 **Actions** 页面可实时查看构建日志；首次运行约 5–15 分钟。
> 新版本发布后，所有已装用户（安装版与便携版 alike）打开软件 3 秒即会弹出更新提示。

#### 备选：本地手动出包（需 Node 22 + 网络）

```bash
# 配置镜像与缓存（国内网络用 npmmirror 拉 Electron 二进制，避免从 GitHub 卡死）
export ELECTRON_MIRROR="https://cdn.npmmirror.com/binaries/electron/"
export ELECTRON_CUSTOM_DIR="{{ version }}"
export ELECTRON_CACHE="D:/xinjing-electron/.electron-cache"
export ELECTRON_BUILDER_CACHE="D:/xinjing-electron/.builder-cache"
export npm_config_cache="D:/xinjing-electron/.npm-cache"
npm install
npm run pack                      # 仅绿色版：dist/win-unpacked/
npm run dist                      # 仅本地出包（不发布）
export GH_TOKEN="你的发布用 GitHub Token（需 contents:write 权限）"
npm run release                   # 本地出包并发布到 Releases
```

### 6.2 用户侧自动更新

- 已安装旧版的用户，程序启动约 3 秒后自动去 GitHub Releases 比对版本（安装版读 `latest.yml`，便携版读 `latest-portable.yml`）。
- 发现新版 → 弹窗「发现新版本」→ 用户点「立即更新」→ 后台下载 → 退出时自动替换。
- **安装版（setup.exe）用户**：走标准 NSIS 更新，原地升级到 `Program Files`。
- **便携版（portable.exe）用户**：读 `latest-portable.yml`，优先走 blockmap 差分增量（只下载变化块），退出时由 detached 批处理把新 exe 覆盖到运行中的便携文件、再重启，**磁盘上始终只有一份**。
- 用户**无需重装、数据不丢**（数据在 `%APPDATA%/XinJing/`，与程序分离）。

### 6.3 安全与令牌提醒

- 发布用的 `GH_TOKEN` 只在你执行 `npm run release` 时通过环境变量传入，**不要写进代码或仓库**。
- 之前 `winnicott-chat/public/.git/config` 里明文存过 PAT，建议抽空用
  `git -C "…/winnicott-chat/public" remote set-url origin https://github.com/mei-junhao/winnicott-chat.git`
  清掉明文；GitHub 后台亦可单独建一个"只用于发布"的 token，别复用推送用的那一个。
- 激活主密钥在 `license-core.js`：分发安装包即包含该密钥（离线校验所必需），仅能挡住普通传播，无法对抗专业逆向；如需更强保护需改用在线授权服务器。

工程在 `D:/xinjing-electron/`，全部依赖与缓存都在 D 盘，**不占用 C 盘系统目录**。

```bash
cd D:/xinjing-electron

# 1) 更新前端代码：把最新的心镜 app 文件覆盖进 app/ 目录
#    （不要带 *.md 文档、tools/ 等开发文件）

# 2) 生成图标（如改了图标）
node generate-icon.js

# 3) 安装/更新依赖（缓存指向 D 盘）
export ELECTRON_CACHE=/d/.electron-cache
export ELECTRON_BUILDER_CACHE=/d/.electron-builder-cache
export npm_config_cache=/d/.npm-cache
npm install

# 4) 本地预览（开发态）
npm start

# 5) 出安装包
npm run dist
# 产物：D:/xinjing-electron/dist/心镜-XinJing-setup-1.0.0.exe
```

版本号修改：编辑 `package.json` 的 `version` 字段，安装包文件名会自动带新版本。

## 七、已知限制

- 安装包体积较大（约 100–160MB），因为内置了完整 Chromium 内核。这是 Electron 方案的固有代价。
- 未代码签名，企业环境若强制 SmartScreen 策略可能需 IT 放行或自行签名。
- 托盘常驻意味着关闭窗口 ≠ 退出进程；确认退出请右键托盘「退出」。
